# Salesforce 集成数据中心页面设计文档

## 1. 概述

本文档详细描述了 Salesforce 集成模块数据中心页面的设计方案，旨在为前端开发提供清晰的页面结构、功能模块和数据展示建议。

### 1.1 页面目标

- 集中展示 Salesforce 集成模块的核心功能和运行状态
- 提供实时监控和管理能力
- 支持数据分析和决策
- 提供告警和异常处理机制

### 1.2 设计原则

- **实时性**：关键指标实时更新，自动刷新机制
- **可视化**：使用图表、进度条、仪表盘等可视化元素
- **可操作性**：支持直接操作，如启动/停止同步、修改配置
- **可筛选**：提供多维度筛选和搜索功能

---

## 2. 页面整体结构

### 2.1 导航菜单

```
数据中心
├── 概览仪表盘
├── 对象管理
├── 批次管理
├── API 监控
├── 日志中心
└── 高级功能
    ├── 实时监控大屏
    ├── 数据分析报表
    └── 告警管理
```

### 2.2 布局设计

- **顶部导航栏**：显示当前页面标题和全局操作按钮
- **侧边菜单**：导航到不同功能模块
- **主内容区**：展示当前模块的具体内容
- **底部状态栏**：显示系统状态和最后更新时间

## 3. 核心模块详细设计

### 3.1 概览仪表盘

#### 3.1.1 核心指标卡片

| 指标名称 | 数据来源 | 显示格式 | 说明 |
|---------|---------|---------|------|
| 总对象数 | DataiIntegrationObject | 数字 | 已配置的 Salesforce 对象总数 |
| 启用同步对象数 | DataiIntegrationObject (isWork=true) | 数字 | 当前启用同步的对象数量 |
| 今日同步批次 | DataiIntegrationBatch (syncStartDate=今天) | 数字 | 今天执行的同步批次数量 |
| 成功率 | DataiIntegrationBatch | 百分比 | 同步成功率 |
| API 调用次数 | DataiIntegrationApiCallLog | 数字 | 今日 API 调用总次数 |
| 限流告警数 | DataiIntegrationRateLimit | 数字 | 当前限流告警数量 |

#### 3.1.2 API 限流状态

显示各 API 类型的限流状态：

- **Bulk API V1**: 可用令牌数、今日已用、每日限额、使用率
- **Bulk API V2**: 可用令牌数、今日已用、每日限额、使用率
- **SOAP API**: 可用令牌数、今日已用、每日限额、使用率
- **REST API**: 可用令牌数、今日已用、每日限额、使用率

**数据来源**: [RealTimeStatusDTO](file:///d:\idea_demo\datai\datai-scenes\datai-scene-salesforce\datai-salesforce-integration\src\main\java\com\datai\integration\domain\dto\RealTimeStatusDTO.java)

**显示格式**: 进度条 + 数字 + 百分比

**颜色标识**:
- 绿色：使用率 < 50%
- 黄色：使用率 50%-80%
- 红色：使用率 > 80%

#### 3.1.3 同步进度监控

显示当前正在进行的同步任务：

- 对象名称
- 同步类型（全量/增量）
- 进度百分比
- 已处理记录数 / 总记录数
- 预计剩余时间
- 状态（运行中/暂停/完成）

**数据来源**: [DataiIntegrationBatch](file:///d:\idea_demo\datai\datai-scenes\datai-scene-salesforce\datai-salesforce-integration\src\main\java\com\datai\integration\domain\DataiIntegrationBatch.java) (syncStatus=true)

- 告警类型（限流/同步失败/连接异常）

## 4. API 接口说明

### 4.1 现有 API 接口

基于 [API Controller 方法清单文档.md](file:///d:\idea_demo\datai\datai-scenes\datai-scene-salesforce\datai-salesforce-integration\src\main\resources\doc\API Controller 方法清单文档.md)，现有接口包括：

#### 对象管理接口
- `GET /objects` - 获取对象列表
- `GET /objects/{id}` - 获取对象详情
- `POST /objects` - 创建对象
- `PUT /objects/{id}` - 更新对象
- `DELETE /objects/{id}` - 删除对象

#### 批次管理接口
- `GET /batches` - 获取批次列表
- `GET /batches/{id}` - 获取批次详情
- `POST /batches` - 创建批次
- `PUT /batches/{id}` - 更新批次

#### API 监控接口
- `GET /api-call-logs` - 获取 API 调用日志
- `GET /rate-limits` - 获取限流配置
- `PUT /rate-limits/{id}` - 更新限流配置

#### 日志接口
- `GET /sync-logs` - 获取同步日志
- `GET /metadata-changes` - 获取元数据变更记录

### 4.2 建议新增 API 接口

#### 概览仪表盘接口
- `GET /dashboard/overview` - 获取概览数据
- `GET /dashboard/api-status` - 获取 API 状态
- `GET /dashboard/sync-progress` - 获取同步进度
- `GET /dashboard/alerts` - 获取告警列表
- `GET /dashboard/api-trends` - 获取 API 调用趋势

#### 统计分析接口
- `GET /statistics/usage` - 获取使用统计
- `GET /statistics/sync-efficiency` - 获取同步效率统计
- `GET /statistics/data-growth` - 获取数据增长统计

#### 实时监控接口
- `GET /monitor/realtime` - 获取实时监控数据
- `WebSocket /ws/monitor` - 实时数据推送

#### 告警管理接口
- `GET /alerts` - 获取告警列表
- `PUT /alerts/{id}/handle` - 标记告警已处理
- `GET /alert-rules` - 获取告警规则
- `POST /alert-rules` - 创建告警规则

| 同步进度 | 10 秒 | 同步进度频繁更新 |
| 告警列表 | 60 秒 | 告警列表定期更新 |
| API 调用趋势 | 5 分钟 | 趋势数据不需要频繁更新 |
| 对象列表 | 手动刷新 | 对象配置不频繁变化 |
| 批次列表 | 手动刷新 | 历史数据不需要频繁更新 |
| API 调用日志 | 手动刷新 | 日志数据量大，手动刷新 |
| 实时监控大屏 | 5 秒 | 全屏监控需要高频刷新 |
| 数据分析报表 | 手动刷新 | 报表数据手动刷新 |

### 5.2 刷新机制

- **前端定时刷新**: 使用 `setInterval` 定时调用 API
- **WebSocket 推送**: 对于实时监控大屏，使用 WebSocket 推送数据
- **手动刷新**: 提供刷新按钮，用户手动触发
- **智能刷新**: 当用户离开页面时暂停刷新，返回时恢复

- **框架**: React / Vue / Angular
- **UI 组件库**: Ant Design / Element UI / Material-UI
- **图表库**: ECharts / Chart.js / D3.js
- **状态管理**: Redux / Vuex / MobX

- **WebSocket**: Socket.io / native WebSocket

### 7.2 后端技术栈

- **框架**: Spring Boot
- **ORM**: MyBatis / MyBatis Plus
- **数据库**: MySQL / PostgreSQL
- **缓存**: Redis
- **任务调度**: Spring @Scheduled / Quartz
- **WebSocket**: Spring WebSocket / Socket.io

### 7.3 性能优化建议

1. **数据缓存**: 使用 Redis 缓存频繁访问的数据
2. **分页加载**: 大数据量列表使用分页加载
3. **懒加载**: 详情页按需加载数据
4. **防抖节流**: 搜索和筛选操作使用防抖节流
5. **虚拟滚动**: 长列表使用虚拟滚动优化性能
6. **CDN 加速**: 静态资源使用 CDN 加速

### 7.4 安全性建议

1. **权限控制**: 基于角色的访问控制（RBAC）
2. **API 认证**: 使用 JWT 或 OAuth2 认证
3. **数据加密**: 敏感数据加密存储
4. **SQL 注入防护**: 使用参数化查询
5. **XSS 防护**: 对用户输入进行转义和过滤
6. **CSRF 防护**: 使用 CSRF Token
